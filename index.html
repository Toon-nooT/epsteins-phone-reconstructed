About This Project

What you're about to see is a fantasy representation of Jeffrey Epstein's phone. The messages displayed are based on email documents officially released by the U.S. House Committee on Oversight and Government Reform on November 12, 2025 (source).

The documents linked in the Documents app were officially released by the U.S. House Committee on Oversight and Government Reform on September 8, 2025 as part of records provided by the Epstein Estate (source).

Who am I?

My name is Toon, and I like data science. When I looked through some samples of the released files, I noticed that the emails sent back and forth were usually one-liners, making it difficult to keep track of the conversations. I thought it would be useful to extract the messages themselves and represent them as if they were sent in a messenger app‚Äîmaking them easier to follow and understand.

For questions, feedback, bug reports, or collaboration inquiries, feel free to reach out at: toon@neontreebot.be

Data Processing & Accuracy

I did my utmost best for the data to stay faithful to the source documents. The messages were extracted using Vision AI (Large Language Models) from screenshot images of the original emails. However, it's inevitable that some mistakes, omissions, or misinterpretations are present. Many messages were redacted and not included.Despite these limitations, I believe this project is valuable because it provides easier access to and navigation of the released files.

When you right-click (desktop) or long-press (mobile) on any message, you have the option to view the original source document it was extracted from‚Äîso you can always verify the information yourself.

üì± About Epstein's Phone (Reconstructed)

A public-interest, independent data project based entirely on official U.S. House Oversight document releases.

üß© What This Project Is

Epstein's Phone (Reconstructed) is a two-part research project:

1. A Structured Message Dataset

A data-science pipeline processed 23,124 images and thousands of OCR text files released by the U.S. House Committee on Oversight and Government Reform in 2025.

Using a combination of OCR cleanup, vision-LLM extraction, name normalization, timestamp reconstruction, and thread mapping, these public images were transformed into a clean, coherent SQLite message database.

This structured dataset contains:

  • sender / recipient fields
  • subjects and thread boundaries
  • message timestamps (raw + normalized)
  • cleaned text + extracted HTML content
  • links back to every original document page

No private data, leaks, or hacked materials were used. Every item in this database originates only from public, government-released files.

2. An iPhone-Style Messages Interface

The second part is an interactive, mobile-first simulation of a phone inbox, designed to make the dataset readable to the public.

Built as a Progressive Web App, the interface provides:

  • an iPhone-style Messages layout
  • contact list, timestamps, and thread separators
  • chat-bubble rendering for senders and recipients
  • long-press context menus
  • links to original document JPGs
  • full offline support via service worker
  • installation like a native app (no backend required)

This front-end is intentionally familiar and intuitive: a way to navigate thousands of messages in seconds, using a format most people already understand.

üéØ Purpose

The Epstein document releases are technically public but practically inaccessible due to their scale, formatting, and lack of structure.

This project aims to:

  • make public data actually accessible
  • support journalists, researchers, lawyers, and the general public
  • reduce the friction of parsing thousands of screenshots
  • demonstrate how independent data science can support transparency
  • encourage accountable, evidence-based reporting

The project itself takes no position on the content of the messages. Its goal is accessibility, verifiability, and readability.

Users are encouraged to cross-check anything important against the original documents.

üß† How the Reconstruction Works

The dataset was built using a multi-stage extraction pipeline:

  1. Image + OCR ingestion of all released pages
  2. Vision-LLM reading of email screenshots
  3. Extraction of sender/recipient metadata, subjects, timestamps, and message bodies
  4. Normalization & cleaning (names, time formats, duplicates)
  5. Thread reconstruction based on temporal + subject structure
  6. SQLite packaging for fast, offline client-side use
  7. Frontend rendering in a Messages-style interface

All processing steps are fully documented in the repository.
